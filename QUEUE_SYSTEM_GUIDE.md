# 📋 任务队列系统使用指南

## ✨ 功能概述

实现了一个**智能任务队列系统**，支持多个任务依次执行，每个任务都能实时监控进度和队列状态。

## 🎯 核心功能

### 1. **任务队列处理**
- 支持多个任务同时上传
- 任务依次执行（不会同时处理两个任务）
- 自动启动后台队列处理线程

### 2. **任务状态管理**
```
pending → queued → processing → completed/error
         (待上传)  (待执行)    (处理中)    (完成/错误)
```

### 3. **队列位置显示**
- 实时显示任务在队列中的位置
- 显示前面有多少个任务在等待
- 显示当前正在处理的任务名称

### 4. **进度恢复**
- 刷新页面自动恢复队列状态
- 任务开始处理时自动恢复历史进度
- 继续监听实时进度更新

## 📱 用户使用流程

### 场景：创建多个任务，依次上传和处理

#### Step 1：创建第一个任务
```
1. 点击"创建任务"
2. 输入任务名（如：案件1分析）
3. 进入上传页面
4. 上传zip文件 → "文件已上传，等待处理队列..."
5. 显示"⏳ 等待中 - 排队位置: 第1个"
6. 任务开始处理，进度实时更新
```

#### Step 2：创建第二个任务（第一个还在处理）
```
1. 返回主页面 or 点击新建任务
2. 创建新任务（如：案件2分析）
3. 上传zip文件
4. 显示"⏳ 等待中 - 排队位置: 第2个"
5. 显示"正在处理: 案件1分析"
6. 等待第一个任务完成
```

#### Step 3：继续创建更多任务
```
1. 可以继续创建第3、4、5...个任务
2. 每个任务都会排队等待执行
3. 可随时刷新页面查看队列状态
```

#### Step 4：查看任务列表
```
主页面会显示：
- 🔄 处理中: 当前正在处理的任务（最多1个）
- ⏳ 待执行: 等待队列中的任务（可能有多个）
- ✅ 已完成: 完成的任务（可重跑或下载）
```

## 🔧 后端实现细节

### 新的API端点

#### 1. POST `/api/process_save/`（上传文件到队列）
```python
请求：form-data {zip_file, task_id}
返回：
{
    "code": 0,
    "msg": "文件已上传，等待处理队列...",
    "task_id": 123
}
```

#### 2. GET `/api/tasks/<task_id>/queue_status/`（获取队列状态）
```python
返回：
{
    "code": 0,
    "status": "queued",
    "queue_position": 2,      # 当前任务在队列中的位置
    "queue_total": 3,         # 总共有3个任务等待
    "processing_task_id": 1,
    "processing_task_name": "案件1分析"
}
```

#### 3. GET `/api/queue/list/`（获取整个队列列表）
```python
返回：
{
    "code": 0,
    "processing": [          # 处理中的任务（最多1个）
        {
            "id": 1,
            "task_name": "案件1分析",
            "status": "processing"
        }
    ],
    "queued": [             # 等待队列（可能多个）
        {
            "id": 2,
            "task_name": "案件2分析",
            "status": "queued"
        },
        {
            "id": 3,
            "task_name": "案件3分析",
            "status": "queued"
        }
    ]
}
```

### 后台队列处理线程

```python
process_task_queue() 线程：
├─ 每1秒检查一次
├─ 获取最早的 queued 任务
├─ 标记为 processing
├─ 执行任务处理
└─ 完成后自动处理下一个任务
```

## ❓ 常见问题

**Q: 为什么我上传的文件后立即显示队列状态？**
A: 这是正常的。队列系统会自动管理任务执行，当轮到你的任务时会自动开始处理。

**Q: 如果我关闭浏览器会怎样？**
A: 后台处理会继续进行。打开浏览器重新进入任务页面，会自动恢复进度。

**Q: 能否加速队列处理？**
A: 不建议。当前设计是依次处理以防止资源竞争。如需并行处理，需要修改队列线程。

**Q: 如何查看整个队列？**
A: 返回主页面，"进行中"标签显示处理中的任务，"已完成"显示完成的任务。

**Q: 能否跳过某个排队任务？**
A: 不能。当前是严格按创建时间顺序处理。可以删除任务从队列中移除。
